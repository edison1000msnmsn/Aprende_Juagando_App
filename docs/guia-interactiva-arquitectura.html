No revelar correctAnswerGET /activities/:id entrega payload e instrucción, pero excluye la respuesta. Flutter propone; el backend valida.
¿Qué hace ValidationPipe?
Transforma y valida DTO. whitelist limita campos permitidos y forbidNonWhitelisted rechaza propiedades inesperadas.
¿Por qué una transacción?
Intento, finalización, estrellas y progreso deben cambiar juntos. Si algo falla, PostgreSQL revierte todo.

09 · Contenido colaborativo

Cómo agregar niveles

Cada nivel es JSON. No se cambia Flutter o NestJS por cada nuevo contenido.

Copia el último nivel del áreaEjemplo: content/mathematics/level-002.json.
Cambia id, número, título y actividadesIDs únicos y números crecientes.
Trabaja inicialmente con DRAFTSolo PUBLISHED llega a la app.
Ejecuta npm run content:validateContrato, IDs, feedback y accesibilidad.
Cambia a PUBLISHED y ejecuta db:seedUpsert a PostgreSQL.
Vuelve a abrir el mundo en FlutterSe carga dinámicamente.

15 · Protección

Implementado y pendiente

Implementado

  • Argon2 para contraseñas.
  • JWT corto + refresh rotativo.
  • Refresh hasheado.
  • Guard JWT y propiedad del perfil.
  • ValidationPipe, Helmet y CORS.
  • correctAnswer oculto.
  • Almacenamiento seguro móvil.
  • Recompensas idempotentes.

Antes de producción

  • HTTPS y secretos administrados.
  • Refresh automático en Dio.
  • Recuperación y verificación de cuenta.
  • Rate limiting.
  • Privacidad, consentimiento y eliminación.
  • Logs, backups y monitoreo.
  • Desactivar cleartext Android.
  • Firma release.
Desarrollo no es producciónLos secretos de Compose son locales. usesCleartextTraffic=true permite HTTP solo para desarrollo.
16 · Diagnóstico

Problemas frecuentes

Flutter no conecta con la API
Comprueba docker compose ps y health. Emulador: 10.0.2.2. Teléfono: IP LAN y firewall.
El nivel no aparece
Carpeta correcta, status PUBLISHED, número creciente, IDs únicos, validador aprobado y npm run db:seed.
Prisma Client no generado
Ejecuta npm run db:generate.
La migración no conecta
Windows usa 55432; la red Docker usa postgres:5432.
401 Unauthorized
El access token venció. La renovación automática Flutter está pendiente; vuelve a iniciar sesión.
Perfil no encontrado
También significa que el perfil no pertenece a esa cuenta. Verifica token y profileId.
Las estrellas no aumentan al repetir
Es correcto: solo el primer acierto premia. Completion y clientAttemptId evitan duplicados.
Android Studio no reconoce el proyecto
Abre exactamente frontend/mobile, ejecuta flutter pub get y revisa flutter doctor -v.
